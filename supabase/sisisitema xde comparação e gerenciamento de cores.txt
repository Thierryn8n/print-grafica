Quero implementar no meu app um sistema de análise e conversão de cores para sublimação, integrado à página de pedido do cliente. O app já existe, então não quero criar um app do zero. Quero adicionar um módulo novo dentro da página de pedido, para ajudar o designer a pegar uma imagem enviada pelo cliente e descobrir quais cores corretas usar no Corel/Pantone para que, depois da sublimação, a camisa fique o mais próxima possível da imagem de referência.

Contexto do sistema:
Eu trabalho com gráfica/sublimação de fardamentos, camisas esportivas e uniformes. O problema é que as cores que aparecem no Corel, RGB ou Pantone digital não saem iguais no tecido depois da sublimação. Geralmente ficam mais escuras, mudam saturação, mudam brilho e podem puxar para outro tom. Além disso, Dry-fit e Helanca dão resultados diferentes. Por isso, quero criar um banco de dados real com amostras escaneadas das cores sublimadas em cada tecido.

Vou usar o Supabase como banco de dados, autenticação e storage.

O sistema precisa funcionar com este fluxo:

1. Na página de pedido do cliente, o cliente ou atendente envia uma imagem de referência.
   Essa imagem pode ser:

* camisa feita por IA;
* mockup;
* foto de uniforme;
* arte pronta;
* imagem de exemplo enviada pelo cliente.

2. O sistema analisa a imagem enviada e extrai as cores principais.
   Não quero que ele pegue todos os pixels como cores separadas, pois isso gera milhares de cores desnecessárias. O algoritmo precisa agrupar cores parecidas e retornar uma paleta organizada com as cores dominantes da imagem.

O algoritmo pode usar:

* K-means;
* Median Cut;
* Color Quantization;
* conversão RGB para LAB;
* comparação por Delta E / CIEDE2000.

O ideal é trabalhar com LAB para comparação visual, porque LAB se aproxima mais da percepção humana do que RGB puro.

3. Depois de extrair as cores principais, o sistema deve comparar cada cor encontrada com um banco de dados real de cores sublimadas.

Esse banco de dados terá:

* cores originais do Corel/Pantone/RGB;
* cores escaneadas depois da sublimação no Dry-fit;
* cores escaneadas depois da sublimação na Helanca;
* código original usado no Corel;
* tecido;
* imagem da amostra;
* RGB medido;
* LAB calculado;
* Delta E;
* observações de produção.

4. O objetivo principal não é apenas simular a cor escura.
   O objetivo principal é responder:
   “Para a cor da imagem do cliente ficar parecida no Dry-fit, qual código Corel/Pantone eu devo usar?”
   e também:
   “Para a cor da imagem do cliente ficar parecida na Helanca, qual código Corel/Pantone eu devo usar?”

12. Algoritmo de análise:

Quando o cliente enviar a imagem:

* carregar a imagem;
* reduzir tamanho para processamento se necessário;
* remover pixels muito transparentes se houver PNG;
* converter para RGB;
* extrair paleta dominante usando K-means ou Median Cut;
* converter cores detectadas de RGB para LAB;
* para cada cor detectada, buscar no banco de dados as cores sublimadas do tecido escolhido;
* calcular Delta E / CIEDE2000 entre a cor detectada e cada cor sublimada cadastrada;
* ordenar pelo menor Delta E;
* retornar a melhor cor correspondente;
* salvar o resultado no Supabase.

13. Importante:
    A comparação deve ser feita entre a cor da imagem do cliente e a cor já sublimada escaneada, não apenas com o RGB original do Corel.

A lógica certa é:
cor desejada do cliente → comparar com cores reais sublimadas → retornar o código original Corel/Pantone que gerou aquela cor sublimada.

14. Exemplo:
    Se o cliente manda uma camisa azul escura.
    O sistema identifica a cor HEX #123A8A.
    Ele converte essa cor para LAB.
    Depois procura no banco Dry-fit qual amostra sublimada ficou mais parecida com esse azul.
    Se a amostra mais parecida foi gerada originalmente pelo código Corel 293 C, o sistema recomenda:
    “Para Dry-fit, use 293 C no Corel.”

Se na Helanca a cor mais próxima foi gerada pelo código 2945 C, o sistema recomenda:
“Para Helanca, use 2945 C no Corel.”

* Delta E 2 a 5: boa;
* Delta E 5 a 10: média;
* Delta E acima de 10: baixa / precisa de teste.
